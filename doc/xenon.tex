    \section{组成部分}
    Xenon 包含 3 个部分：
    \begin{enumerate}
        \item 预处理器；
        \item 前端；
        \item 后端。
    \end{enumerate}
    \section{预处理器}
    按照顺序，预处理器负责：
    \begin{enumerate}
        \item 将单独的 \verb|\r|，或 \verb|\r\n| 替换为 \verb|\n|。
        \item 对于所有的以反斜杠结尾的行，将其与下一行连在一起. 例如：
        \item 删除所有的注释。
    \end{enumerate}
    以上三个步骤各通过一个自动机完成。自动机被组织成生成器的形式（使用 create generator）。
    \section{前端}
    前端负责将不包含注释的 SysY 源代码翻译为 KoopaIR。前端的处理步骤为：
    \begin{enumerate}
        \item 使用 create pest 将代码处理为语法树。注意，这里语法树使用的是 pest 中 \verb|Pairs<Rule>| 的形式。

        使用 pest 的原因是 pest 基于解析表达式文法，可以轻松绕过悬垂 else 问题。

        注意，只使用 pest parser 难以处理运算符优先级。在 pest parser 的基础上，表达式又会经过 PrattParser<Rule> 的解析。
        \item 遍历上述 \verb|Pairs<Rule>| 的语法树，转为我定义的抽象语法树。这一步使用大量的递归函数。
        \item 遍历抽象语法树，进行语义检查和常量表达式计算。这一步使用大量的递归函数。

        实际实现中，2、3 步是合并在一起的。除此之外，还会进行以下两个化简：
        \begin{enumerate}
            \item 表达式化简，如 w = (x - x) * (y + z); 会直接化简为 w = 0;
            \item （简易的）不可达语句消除。任意 return 后的语句，或是循环内 break/continue 后的语句，都会直接忽略。
        \end{enumerate}

        为了生成 IR 时更加方便，还会进行两个操作：
        \begin{enumerate}
            \item 在初始化列表中填充 0；
            \item 名字重整。
        \end{enumerate}
        \item 生成文本形式的 IR。同样地，这一步使用大量的递归函数。

        SysY 中的表达式被进一步分为三类：
        \begin{enumerate}
            \item 左值表达式，这种表达式计算的值是一个内存地址，如语句 x = 1; 中的表达式 x 是左值表达式；鉴于 Xenon 对 SysY 做了语法扩展，语句 (x = y) = z; 中的表达式 x 和 x = y 都是左值表达式。
            \item 右值表达式，这种表达式计算的值会被舍弃
            \item 弃值表达式，这种表达式计算的值会被舍弃。如语句 x = 1; 中的 x = 1 是弃值表达式，这个语句只会执行赋值的副作用而不产生值；
        \end{enumerate}

        例如，语句

(x = y) = (z = w);

被翻译为 IR：
        \begin{verbatim}
%3 = load %_Iw
store %3, %_Iz
%4 = load %_Iy
store %4, %_Ix
store %3, %_Ix
        \end{verbatim}
        在这之中，表达式 z = w 是右值表达式，x = y 是左值表达式，而 (x = y) = (z = w) 整体是弃值表达式。

        三类表达式都有专门的函数处理。
    \end{enumerate}
