\section{语言扩展}
除标准 SysY 语言外，ATC-X 扩展了如下功能：
\begin{enumerate}
    \item 二进制整数字面量，如：
    \begin{minted}[linenos,frame=single]{c}
int a = 0b11011111101010010; // a = 114514
    \end{minted}
    \item 自定义中缀运算符，例如：
    \begin{minted}[linenos,frame=single]{c}
int foo(int x) { /* do something */ }
int foo2(int x) { /* do something */ }
int bar(int x, int y) { /* do something */ }
int main() {
    int x = 2, y = 2;
    x ~bar~ y;            // 等价于 bar(x, y);
}
    \end{minted}
    \item 管道运算符与成员访问运算符，例如：
    \begin{minted}[linenos,frame=single]{c}
int foo(int x) { /* do something */ }
int foo2(int x) { /* do something */ }
int bar(int x, int y) { /* do something */ }
int main() {
    int x = 2, y = 2;
    x |> foo |> foo2;     // 等价于 foo2(foo(x));
    x |> foo() |> bar(y); // 等价于 bar(y, foo(x));
    bar(x) <| y;          // 等价于 bar(x, y);
    ((x.bar(y)) ~bar~ 114)
      |> foo;             // 等价于 foo(bar(bar(x, y), 114));
}
    \end{minted}
    \item 函数声明语法，允许在函数签名中不指明参数名称，对于没有参数的函数也允许使用
    \inlinec{void} 为参数列表。例如：
    \begin{minted}[linenos,frame=single]{c}
int foo(int);
int bar(int, int);
int main(void) { return foo(bar(114, 514)); }
int bar(int x, int) { return x + 1919; }
    \end{minted}
    \item 各类位运算符，如 \inlinec{<<}、\inlinec{^} 和 \inlinec{~}。
    \item 各类复合赋值运算符，如 \inlinec{+=}，\inlinec{*=}。
    \item 赋值运算符返回左值，这意味着可以写 \inlinec{x = y = z;} 这种代码。
    \item 自增自减运算符，其中前缀自增自减返回左值。例如 \inlinec{++x = 1;}
    \item 以常量下标取得的常量数组元素可以参与编译期常量表达式求值。
\end{enumerate}
\section{组成部分}
ATC-X 包含 3 个部分：
\begin{enumerate}
    \item 预处理器；
    \item 前端；
    \item 后端。
\end{enumerate}
\section{预处理器}
按照顺序，预处理器负责：
\begin{enumerate}
    \item 将单独的 \inlinec{\r}，或 \inlinec{\r\n} 替换为 \inlinec{\n}。
    \item 对于所有的以反斜杠结尾的行，将其与下一行连在一起。例如：
    \begin{minted}[linenos,frame=single]{c}
int main() {
    int a = 1; // \
    int a;
    return a;
}
    \end{minted}
    被处理为：
    \begin{minted}[linenos,frame=single]{c}
int main() {
    int a = 1; //     int a;
    return a;
}
    \end{minted}
    \item 删除所有的注释。
\end{enumerate}
以上三个步骤各通过一个自动机完成。自动机被组织成生成器的形式（使用 crate genawaiter）。
\section{前端}
前端负责将不包含注释的 SysY 源代码翻译为 KoopaIR。前端的处理步骤为：
\subsection{第一次解析}
使用 crate pest 将代码处理为语法树。注意，这里语法树使用的是 pest 中 \inlinerust{Pairs<Rule>} 的形式。
使用 pest 的原因是 pest 基于解析表达式文法，可以轻松绕过悬垂 else 问题。注意，只使用 pest parser
难以处理运算符优先级。在 pest parser 的基础上，表达式又会经过 \inlinerust{PrattParser<Rule>} 的解析。
\subsection{第二次解析}
遍历上述 \inlinerust{Pairs<Rule>} 的语法树，转为我定义的抽象语法树。这一步使用大量的递归函数。
\subsection{语义检查}
遍历抽象语法树，进行语义检查和常量表达式计算。这一步使用大量的递归函数。

实际实现中，第二次解析和语义检查是合并在一起的。除此之外，还会进行表达式化简，如
\inlinec{w = (x - x) * (y + z);} 会直接化简为 \inlinec{w = 0;}。

为了生成 IR 时更加方便，还会进行两个操作：
\begin{enumerate}
    \item 在初始化列表中填充 0；
    \item 名字重整，即为每个标识符赋予一个局部或全局唯一的名字。对局部常量数组有特殊处理，因为生成 IR
    时局部常量数组会被放至全局。
\end{enumerate}

代码：
\begin{minted}[linenos,frame=single]{c}
int f(int a[]) {
    const int x[1] = {1};    int y = 1, z[1] = {1};
    { int x = x[0], y = x; } }
int g(int a[]) {
    const int x[1] = {1};    int y = 1, z[1] = {1};
    { int x = x[0], y = x; } }
\end{minted}
被重整化为：
\begin{minted}[linenos,frame=single]{c}
int f(int _Pa[]) {
    const int _Ax[1] = {1};    int _Iy = 1, _Az[1] = {1};
    { int _Ix = _Ax[0], _I_Iy = _Ix; } }
int g(int _Pa[]) {
    const int _A_Ax[1] = {1};    int _Iy = 1, _Az[1] = {1};
    { int _Ix = _A_Ax[0], _I_Iy = _Ix; } }
\end{minted}
考虑到 C 标准中将“下划线+大写字母”开头的标识符视为保留名，这样做是完全合理的。
\subsection{生成 IR}
\subsubsection{生成粗 IR}
前端直接生成文本形式的 IR。同样地，这一步使用大量的递归函数。

SysY 中的表达式被进一步分为三类：
\begin{enumerate}
    \item 左值表达式，这种表达式计算的值是一个内存地址，如语句 \inlinec{x = 1;} 中的表达式
    \inlinec{x} 是左值表达式；
    鉴于 ATC-X 对 SysY 做了语法扩展，语句 \inlinec{(x = y) = z;} 中的表达式 \inlinec{x}
    和 \inlinec{x = y} 都是左值表达式。
    \item 右值表达式，这种表达式计算的值只是一个值。
    \item 弃值表达式，这种表达式计算的值会被舍弃。如语句 \inlinec{x = 1;} 中的 \inlinec{x = 1} 是
    弃值表达式，这个语句只会执行赋值的副作用而不产生值；
\end{enumerate}
\subsubsection{IR 后处理}
上述生成的 IR 称为粗 IR，因为它有很多不符合规范的情况。例如，代码：
\begin{minted}[linenos,frame=single]{c}
void f() { }
int main() {
  { while(1)
      continue;
    return 0; }
  return 0;
}
\end{minted}
会生成 IR：
\begin{minted}[linenos,frame=single]{c}
fun @f() {
%1:
}
fun @main(): i32 {
%3:
    jump %6
%6:
    jump %6
    jump %6
%7:
    ret 0
    ret 0
}
\end{minted}

可以看到，某些基本块存在连续的 \inlinec{jump}，某些基本块是空的。后处理就是要处理这些不合法的 IR，
分为两个部分：
\begin{enumerate}
    \item 基本块内部，第一个 \inlinec{jump}、\inlinec{br} 或 \inlinec{ret} 后的所有指令被忽略；
    \item 如果某个基本块的结尾不是 \inlinec{jump}、\inlinec{br} 或 \inlinec{ret}，则根据函数
    返回值类型添加 \inlinec{ret 0} 或 \inlinec{ret}。
\end{enumerate}

上述 IR 后处理的结果为：
\begin{minted}[linenos,frame=single]{c}
fun @f() {
%1:
    ret
}
fun @main(): i32 {
%3:
    jump %6
%6:
    jump %6
%7:
    ret 0
}
\end{minted}
\section{后端}
后端非常池沼，总的来说就是 \inlinec{t0} \~{} \inlinec{t3} 这几个寄存器来回倒腾；大部分东西都存在栈上，
通过 \inlinec{sp} 访问；只有函数调用或者返回的时候才会碰一碰 \inlinec{a0} \~{} \inlinec{a7}。

在生成汇编之后还有后处理步骤，即：
\begin{itemize}
    \item 如果 \inlinec{j label} 之后恰好就是 \inlinec{label}，则 \inlinec{j} 可以消除；
    \item 消除了冗余的 \inlinec{lw}；
    \item 对于偏移量不在 $-2048$ \~{} $2047$ 范围内的 \inlinec{j}、\inlinec{beqz} 或 \inlinec{bnez}，
    使用 \inlinec{jr} 中转.
\end{itemize}

我不认为后端有介绍的必要。这部分就略过吧。
